Custom Code Action: SpeedRegistry

Classname: com.zerog.ia.customcode.win32.registry.SpeedRegistry

Description:This class will add values to the registry. It reads values from a text file located in the resource file.

Input InstallAnywhere Variables:

"CC_DEBUG": if this IA Variable is set, the action will produce additional debugging information to standard out (optional).

General Instructions:

This help file explains how to set up and take advantage of the SpeedRegistry custom code action.

To effectively use this action, add an "Execute Custom Code" action to the Install Task in the Advanced Designer.   Adding this action to the Install Task will ensure that the uninstall method is called. 

Once you have added this action to your project, select the JAR or ZIP file that contains the desired action. To select the JAR or ZIP file, click the "Choose JAR or ZIP..." button, and select the customcode.jar file in the "Choose Code Archive" dialog that appears.

Next, specify the fully qualified main class in the "Class:" text field.  In this case, you would enter "com.zerog.ia.customcode.win32.registry.SpeedRegistry" (no quotes).
 
Win32 registry entries will be made by the SpeedRegistry custom code. The registry entries are read from a plain text file called "registry.txt" located in the "resource" directory of the "registry" directory of the JAR or ZIP file.  For details on this file and how to create or modify entries included in this file, see the included example file in the "resource" directory of the "registry" directory of the JAR or ZIP file.
