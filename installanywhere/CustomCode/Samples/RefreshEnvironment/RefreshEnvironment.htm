Custom Code Action: RefreshEnvironment

Classname: RefreshEnvironment

Description: This action will force the Windows operating system to refresh it's environment.  This is mainly helpful right after a Set Environment Variable action.

Input InstallAnywhere Variables: none

Output InstallAnywhere Variables: none

General Instructions:

This action is displays how to call a native method within dll from within a custom code action.

This README explains how to set up and take advantage of the RefreshEnvironment custom code action.

To effectively use this action, add an "Execute Custom Code" action in the Advanced Designer.   Once you have added this action to your project, select the JAR or ZIP file that contains the desired action. To select the JAR or ZIP file, click the "Choose JAR or ZIP..." button, and select the customcode.jar file in the "Choose Code Archive" dialog that appears.

Next, specify the fully qualified main class in the "Class:" text field.  In this case, you would enter " RefreshEnvironment " (no quotes).
