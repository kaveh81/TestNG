Uninstaller for Merge Modules and Multiple Products

InstallAnywhere 8.0

When developing an installer that installs multiple products (or uses merge modules), it is important to consider how the uninstallers for these products will integrate. Each uninstaller is tied to a certain product. It does this through its Product ID (found in the Project | Description task). There are three options here: you can decide to have a separate uninstaller for each product, one uninstaller with uninstallation logic for all of products, or produce a master uninstaller that executes the other sub-uninstallers.

Separate Uninstallers: This is the easiest option. Make sure that every product has a unique product ID, and that the uninstallers install to unique locations.

One Uninstaller for Multiple Products (or Merge Modules): InstallAnywhere has the ability to merge the uninstallation logic from separate uninstallers. In order to have one merged uninstaller function for a group of separate products, it is necessary that every product have the same Product ID. Each separate product should then be "demoted" to a feature of the master product. The uninstallers for these separate projects must also share the same uninstaller name and uninstaller location. Every product that installs features will register its features with the uninstaller.

An example:

Product: Acme Office Suite (ID: 97338341-1ec9-11b2-90e2-a43171489d33)

Installer 1: Acme Word Processor and Acme Spreadsheet

Product ID: 97338341-1ec9-11b2-90e2-a43171489d33

Features: Acme Word Processor and Acme Spreadsheet

Installer 2: Acme Slide Show

Product ID: 97338341-1ec9-11b2-90e2-a43171489d33

Features: Acme Slide Show

End Result: Acme Office Suite uninstaller for Acme Word Processor, Spreadsheet, and Slide Show

Master Uninstaller which Executes Sub-Uninstallers: This option is most similar to concept of merge modules used in the installer. Developers use the Execute Uninstaller action to cause the master uninstaller to execute other uninstallers during uninstallation. The advantage of this technique is that is allows developers to seperate their uninstallation logic. Developers who use this option will find the InstallAnywhere Variables $NEVER_UNINSTALLS_VM$ and $REGISTER_UNINSTALLER_WINDOWS$ useful.
