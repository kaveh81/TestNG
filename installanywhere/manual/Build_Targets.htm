Build Targets

InstallAnywhere 8.0

The Build Targets tab defines which target platforms to build the installer. Installers may be created either with a developer selected Java virtual machine bundled with it, or without a Java VM. To build for a set of platforms, enable the combination of platforms desired, and select which VMs to bundle.

Important

If you are building an installer that is targeting the 32-bit and 64-bit portions of a 64-bit Windows system, you do not need to select the 64-bit Java VM. The 32-bit Java VM targets both portions of the registry.

Task

To create an installer for a flavor of Unix that is not in the list of platforms:

  1. In the text field below the Solaris option, type the name of the Unix flavor.
  2. Select a VM from the list to bundle with the new Unix flavor. This text field may also be used to differentiate between two variations of the same installer.

The Download Additional VM Packs takes brings up the VM packs section of Macrovision's Web site. VM packs are stored as .vm (zip/jar) files, which contain the archive of a Java VM and a "vm.properties" file. These VM packs need to be stored as resources in the ../resources/installer_vms directory. The selected bundled VM will be saved on a project by project basis.
