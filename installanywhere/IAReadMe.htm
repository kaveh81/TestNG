Introduction

InstallAnywhere is the solution of choice among multiplatform installation developers worldwide. It delivers the ideal combination of ease-of-use and powerful functionality for creating professional multiplatform installations in minutes. InstallAnywhere is available in Standard and Enterprise editions to meet your specific needs. InstallAnywhere allows you to:

  • Create one installer project for multiple platforms and multiple languages
  • Deploys software onto any client or server platform
  • Allows you to build a multiplatform installer in minutes with the project wizard
  • Includes all of the platform-specific functionality you need to simplify your software deployment

System Requirements

For Systems Running InstallAnywhere (Authoring Environment)

  • 128 MB free RAM
  • Minimum of 8-bit color depth (256 colors)
  • Minimum 1024 x 768 resolution

InstallAnywhere runs on the latest versions of these operating systems, fully updated with the most recent patches and service packs:

  • Windows Server 2003, XP, 2000, NT
  • Mac OS X, Mac OS X Server
  • Solaris/SPARC
  • Linux/Intel: Red Hat, SuSE
  • HP-UX
  • AIX 5.2 or higher

NOTE: Language-localized versions of InstallAnywhere Enterprise Edition (for English, French, German and Japanese developers) are available on the Windows platform only. Installers can be built on any platform for any other platform or language. Localizations for 29 languages are included.

For Target Systems (Installer Environment)

  • 64 MB of free RAM
  • Minimum of 8-bit color depth (256 colors)
  • Minimum 640 X 480 screen resolution

Installers run on any version of these operating systems, as long as the operating system supports Java:

  • Windows Server XP, 2003 (Itanium 2, x86, AMD-64)
  • Windows Vista, 2000, NT, Me, 98
  • Linux/Intel: Red Hat, Red Hat Enterprise Server, UnitedLinux, TurboLinux, SUSE & others
  • Linux for PowerPC, AMD-64, and Itanium 2
  • Mac OS X (PPC & Intel)
  • Solaris SPARC, AMD-64, Intel
  • HP-UX (Itanium 2 & PA-RISC)
  • Tru64, FreeBSD
  • AIX
  • z/OS
  • NetWare
  • Other Unix platforms

NOTE: InstallAnywhere installers are not supported on beta or on early access releases.

Supported Java Virtual Machines

  • Sun: 1.3.x, 1.4.x, 1.5.x, 1.6.x
  • IBM: 1.3.x, 1.4.x, 1.5.x
  • Apple: 1.3.x, 1.4.x, 1.5.x
  • HP: 1.3.x, 1.4.x, 1.5.x

The InstallAnywhere installer bundles Java 1.5 by default for all platforms. Any Java virtual machine can be bundled with an installer ensuring that the target system meets the minimum requirements for both the installers and your applications. To download these bundled VM packs, see InstallAnywhere Downloads.

NOTE: InstallAnywhere installers are not supported on beta or on early access releases of Java.

New Features

InstallAnywhere continues to deliver the functionality developers need to build successful multiplatform installations. InstallAnywhere 8.0.1 installers are now supported on Windows Vista systems and on Sun 1.6.x JVMs.

Bug Fixes

The following fixes and improvements have been made since the release of InstallAnywhere 8.0.0.

  • IOA-000030413: The "Set System Environment Variable" action now works on Windows Vista.
  • IOA-000027560: Projects that build both web-installer and cd-rom distributions for HP-UX no longer fail to build.
  • IOA-000057176: A flaw in the security of the installation script has been addressed in this hotfix.
  • IOA-000029919: A GUI issue related to the SetEnvironmentVariable action has been resolved.
  • IOA-000029119: $WIN_COMMON_DESKTOP$ now resolves correctly for shortcuts on Windows Vista systems.
  • IOC-000057327: Rules added to MagicFolders now work again at run time.
  • IOA-000027741: Debug message "Command.run(): process completed before monitors could start" no longer shows up in console.
  • IOC-000058165: Installers that use a Check Platform rule with Windows Vista as the only platform now install properly on Windows Vista machines.
  • IOA-000028136: Install debug output is now properly redirected to StdErr instead of the console.
  • IOA-000027421: Fixed issue that occurred when importing merge modules containing empty Folders or SpeedFolders.
  • IOC-000056955: Fixed "User Interface Mode Not Supported" error that would periodically occur when using IBM JVM 1.5.

Opening InstallAnywhere

Use the instructions for the operating system on which you are running InstallAnywhere.

  • To open InstallAnywhere on Windows:
    On the Windows start menu, point to Programs then Macrovision and click InstallAnywhere 8 Enterprise or InstallAnywhere 8 Standard. You can also double-click InstallAnywhere.exe, located by default in C:\Program Files\Macrovision\InstallAnywhere 8 Enterprise or C:\Program Files\Macrovision\InstallAnywhere 8 Standard.
  • To open InstallAnywhere on Mac:
    Double-click InstallAnywhere in the Applications/Macrovision/InstallAnywhere 8 Enterprise or Applications/Macrovision/InstallAnywhere 8 Standard folder. You can also double-click the InstallAnywhere alias that is created by default in your dock.
  • To open InstallAnywhere on UNIX, including Solaris, Linux, HP-UX, AIX, and others:
    Type InstallAnywhere from your Macrovision/InstallAnywhere_8_Enterprise or Macrovision/InstallAnywhere_8_Standard folder. You can also run the InstallAnywhere symbolic link that is installed by default into your home folder.

Uninstalling InstallAnywhere

Use the instructions for the operating system on which you are running InstallAnywhere.

  • To uninstall InstallAnywhere from Windows:
    Double-click the Uninstall shortcut, located by default in C:\Program Files\Macrovision\InstallAnywhere 8 Enterprise or C:\Program Files\Macrovision\InstallAnywhere 8 Standard.
  • To uninstall InstallAnywhere from Mac:
    Double-click Uninstall in the Applications/Macrovision/InstallAnywhere 8 Enterprise or Applications/Macrovision/InstallAnywhere 8 Standard folder.
  • To uninstall InstallAnywhere from UNIX, including Solaris, Linux, HP-UX, AIX, and others:
    Type Uninstall from your Macrovision/InstallAnywhere_8_Enterprise or Macrovision/InstallAnywhere_8_Standard folder.
